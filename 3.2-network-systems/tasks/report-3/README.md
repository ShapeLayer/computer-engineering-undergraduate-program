第03回 レポート課題 1/30 17:00
課題はそれぞれ別ファイルにすること。先に一番下の注意事項も読むこと。
03-1. 簡単なメッセージ(文字)を送受信して終了するだけのクライアントとサーバプログラムについて調べ、どのようにコーディングするかまとめなさい。実際に動作するかも試すこと。
クライアント、サーバどちらから通信を始めても良い。
基本的に既存のWebサーバ とWebクライアント (ブラウザなど) を使う以外の方法で実装すること。WebサーバとWebクライアントを自作する場合は良い。
通信プロトコルとして TCP/IP (or UDP/IP) を利用するプログラムとすること。
メッセージの受信とそれに対する応答の送信を行うこと。送受信メッセージはプログラム内に文字を埋め込み、または手動文字入力するのどちらでも良い。
サーバとクライアントが同一のPC上にあってもかまわない。
コードの簡単な解説(どこで何をやっているか)も行うこと。
プログラミング言語は指定しませんが言語名は記述すること。
独自の追加機能を付加してもらっても構いません(加点する場合があります)。
実際に動作を試して動作結果を記載すること。
動作しなかった場合は理由の考察・試したことも含めて報告すること。
動作しなかったこと自体での減点等ありません。
　※ ページ数の指定は特にありません。

03-2. 自分のパソコンにDocker環境を構築し、簡単なWebサーバを立ち上げ動作を確認しなさい。また、その導入・確認の過程をレポートしなさい。

すでに導入済みの人は導入した理由や動いている物などを解説すること。
再度導入し直す必要はありません、ちゃんと動作していないようなら再導入すること。
Docker は導入済みだがWebサーバが未導入の場合は、導入して報告すること。
Docker環境を共有できる状態(こちらで動作確認できる)にする必要はありません。
ちゃんと動作しなかった場合は理由の考察・試したことも含めて報告すること。
動作しなかったこと自体での減点はありません。
レポート作成後の注意: Docker が予期せず動作したままになる事があるので、不要なら必ず停止すること。
　※ ページ数の指定は特にありません。


注)・指示に従っていない場合、減点することがある。・文書形式、スライド形式のどちらでもよい。

## 제03회 리포트 과제 1/30 17:00
과제는 각각 별도 파일로 하는 것. 먼저 맨 아래의 주의사항도 읽는 것.
03-1. 간단한 메시지(문자)를 송수신하고 종료할 뿐의 클라이언트와 서버 프로그램에 대해서 조사해, 어떻게 코딩하는지 정리해 주세요. 실제로 작동할 수도 있습니다.
클라이언트와 서버 모두에서 통신을 시작할 수 있습니다.
기본적으로 기존 웹 서버와 웹 클라이언트(브라우저 등)를 사용하는 것 이외의 방법으로 구현하는 것. 웹 서버와 웹 클라이언트를 자작하는 경우는 좋다.
통신 프로토콜로서 TCP/IP(or UDP/IP)를 이용하는 프로그램으로 하는 것.
메시지를 수신하고 그에 대한 응답을 전송하는 것. 송수신 메시지는 프로그램에 문자를 포함하거나 수동 문자를 입력 할 수 있습니다.
서버와 클라이언트가 같은 PC에 있어도 상관없다.
코드의 간단한 해설(어디서 무엇을 하고 있는가)도 실시하는 것.
프로그래밍 언어는 지정하지 않지만 언어명은 기술하는 것.
독자적인 추가 기능을 부가해도 상관없습니다 (가점하는 경우가 있습니다).
실제로 동작을 시도하여 동작 결과를 기재한다.
동작하지 않았을 경우는 이유의 고찰·시험한 것을 포함해 보고하는 것.
동작하지 않았던 것 자체에서의 감점등이 없습니다.
　※ 페이지수 지정은 특별히 없습니다.

03-2. 자신의 PC에 Docker 환경을 구축해, 간단한 Web 서버를 기동 동작을 확인해 주세요. 또, 그 도입·확인의 과정을 보고해 주세요.

벌써 도입이 끝난 사람은 도입한 이유나 움직이고 있는 물건등을 해설하는 것.
다시 도입할 필요는 없습니다. 제대로 작동하지 않는 것 같으면 다시 도입하는 것.
Docker는 설치되었지만 웹 서버가 배포되지 않은 경우 배포하고보고해야합니다.
Docker 환경을 공유할 수 있는 상태(여기서 동작 확인할 수 있다)로 할 필요는 없습니다.
제대로 동작하지 않았을 경우는 이유의 고찰·시험한 것을 포함해 보고하는 것.
작동하지 않은 것 자체에 감점이 없습니다.
보고 후 주의: Docker가 예기치 않게 작동한 상태로 유지될 수 있으므로 불필요한 경우 반드시 중지해야 합니다.
　※ 페이지수 지정은 특별히 없습니다.
