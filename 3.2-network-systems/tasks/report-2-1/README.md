# 제02회 레포트 과제 그 1 (01/09 17:00 마감)
1. 과제 내용
본 수업에서도 일부 설명했습니다만, IPA(정보처리추진기구)에서 공개하고 있는 「안전한 웹사이트를 만드는 방법(安全なウェブサイトの作り方)」 신판(개정 제7판) PDF의 내용을 설명하는 90분 수업을 상정한 슬라이드를 작성하십시오.
※ 참고 URL: https://www.ipa.go.jp/security/vuln/websecurity/about.html

수업 대상: 학부 3학년생 상정
필수 포함 사항: 본인의 감상 등 (마지막 요약 슬라이드 등에 간단히 기재해도 무방함)
주의사항:

제출 형식: PowerPoint(.pptx) 파일 또는 PDF(.pdf) 파일로 제출할 것.
파일명 규칙: 학번-네트워크시스템레포트과제2-1.pptx (또는 .pdf)
※ 학번은 본인의 번호로 바꿀 것 (숫자 및 하이픈-은 반각 문자 사용).
구성: 1페이지는 반드시 타이틀 슬라이드로 구성할 것.
인용: 도표를 사용해도 좋으나, 문장을 포함하여 인용한 경우에는 인용처를 참고문헌으로 명시할 것 (슬라이드 마지막 등).
분량: 페이지 수는 기준일 뿐이므로 초과해도 무방함. (PDF로 제출할 경우 1페이지당 1슬라이드가 되도록 설정)
제출 기한: 1월 09일(금) 17:00까지.
[참고] 슬라이드 작성을 위한 IPA 가이드 주요 내용 요약
슬라이드 구성 시 활용하실 수 있도록, 해당 URL에 기재된 자료의 핵심 구성을 정리해 드립니다.

제1장: 웹 애플리케이션의 보안 구현
SQL 인젝션, OS 커맨드 인젝션, 크로스 사이트 스크립팅(XSS), CSRF, 세션 관리 미흡 등 11가지 주요 취약점 설명.
각 취약점의 근본적인 해결책 및 영향 저감 대책 제시.
제2장: 웹사이트 안전성 향상을 위한 노력
웹 서버 운영 및 패스워드 취급 방법 등 운영 측면의 보안 대책(7개 항목).
제3장: 실패 사례
실제 소스 코드 예시를 통해 취약점이 발생하는 원인과 수정 예시 설명.
별책 부록:
「안전한 SQL 호출 방법」: 프로그래밍 언어별 안전한 구현 방법.
「웹 건강진단 사양」: 자가 진단을 위한 13개 검사 항목.
학부 3학년 대상 팁:
이미 기초적인 프로그래밍과 네트워크 지식이 있는 상태이므로, 단순히 "위험하다"는 설명보다는 **"왜 이런 코드가 위험한지(원리)"**와 **"어떻게 방어 코드를 작성해야 하는지(실습적 관점)"**를 중심으로 구성하는 것이 좋습니다.
