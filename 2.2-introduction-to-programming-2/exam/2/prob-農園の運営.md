# 문제 1 (농원 운영)

요소 수 100의 배열로 농원을 표현한다.
`char farm[100]`을 만들고 초기 상태는 모든 칸이 `.` 이다.
농원은 하루에 한 번 활동하며, 총 10일의 활동을 기록한다.

활동 종류:

- 물 주기 `w` (water)
- 비료 주기 `f` (fertilize)
- 씨앗 뿌리기 `s` (sow seeds)
- 수확 `h` (harvest)

각 활동마다 적용 범위(start, end)를 입력한다. 예: `0~72`, `52~99`.

## 농원 상태 규칙

- 초기 상태 `.` 에 씨앗 뿌리기(`s`)를 하면 새싹 `^`가 된다.
- 다른 상태에 `s`를 해도 상태는 변하지 않는다.
- 새싹 `^`에 물 주기(`w`)를 누적 3회 하면 채소 `@`가 된다.
- 새싹 `^`에 비료(`f`)를 하면 물 주기 2회로 계산한다.
- 이미 물을 1회 이상 준 새싹이라면 `f` 1회로 채소 `@`가 된다.
- 채소 `@`는 3일이 지나면(3일째) 시든 채소 `*`가 된다.
- 채소 `@` 또는 시든 채소 `*`에 수확(`h`)을 하면 초기 상태 `.`로 돌아간다.

위 상태 전이 로직은 초기 코드에 구현되어 있다.

## 수지(자금) 규칙

초기 자금은 10000원.
하루 단위로 지출/수입을 계산한다.

활동 비용:

- 비료 `f`: 칸당 150원
- 물 주기 `w`: 칸당 50원
- 씨앗 뿌리기 `s`: 칸당 5원

주의:

- 비용은 지정 범위의 모든 칸에 대해 발생한다.
- 실제 상태 변화가 없더라도, 지정 범위에 활동을 수행한 만큼 비용이 든다.

수확 수익:

- `@`: 칸당 500원
- `*`: 칸당 10원

활동 중 자금이 부족하면 그 시점에서 해당 일의 활동을 중단한다.

- 일부만 수행 가능할 때:
  - `자금 부족: <활동명>은 X~Y까지만 수행`
- 한 칸도 수행하지 못할 때:
  - `자금 부족: <활동명>을 수행할 수 없었습니다`

활동은 왼쪽 칸부터 우선 처리한다.

위 규칙에 맞게 초기 코드를 수정해, 일별 수지 계산이 정확히 되도록 하라.

## 입력

각 Day마다 다음 형식 1줄:

`<activity> <start> <end>`

- activity: `s`(씨뿌리기), `w`(물), `f`(비료), `h`(수확)
- start, end: 처리 구역 번호(0~99)

## 기대 출력

시작 시:

- 농원 시뮬레이션 시작(10일간)
- 초기 잔고 : 10000원

매일 출력:

- Day n 활동 입력 안내
- 범위(start end) 안내
- 농원 상태(100문자)
- 잔고
- Day n 수지: 지출 xx원, 수입 yy원 -> 잔고 zz원
- 구분선

종료 시:

- 10일 종료! 최종 잔고 : zz원
