# 문제 8: 크리스마스 트리 오너먼트 관리

완전 이진트리 형태의 크리스마스 트리를 5년 동안 유지보수한다.
각 노드에는 정수 `value`가 저장된다.

- `0~5`: 장식이 설치되어 있고, 값은 설치 후 경과 연수
- `9`: 철거(빈 자리)

초기 트리 값은 제공 코드에서 자동 생성되며, 이 초기화 로직은 수정하지 않는다.

## 연도별 처리(1년차 ~ 5년차)

각 해마다 아래 순서로 처리한다.

1. **경과 연수 증가(2년차부터)**
- 2~5년차에는 `value`가 `0~5`인 노드에 대해 `value += 1`
- `value=9`는 그대로 유지
- 1년차에는 이 단계 없음

2. **해당 연도 추가 개수 입력**
- 입력으로 그 해 추가할 장식 개수 `K`를 받는다. (`K >= 0`)

3. **철거 처리**
- `value > 5`인 노드를 모두 `9`로 변경

4. **장식 추가**
- `value=9`인 빈 노드에 새 장식을 넣고 `value=0`으로 설정
- 배치 우선순위:
	- 레벨이 낮은 노드(루트에 가까운 노드) 우선
	- 같은 레벨에서는 **오른쪽 우선**
	- 즉, 레벨 순서 탐색 시 `오른쪽 -> 왼쪽` 순서
- 빈 노드 수가 부족하면 가능한 만큼만 추가

5. **그 해 결과 출력**
- 트리를 레벨 순서로 출력

## 입력

```text
D
add_count_year1
add_count_year2
add_count_year3
add_count_year4
add_count_year5
```

- `D`: 트리 깊이(루트 깊이=1)
- `1 <= D <= 8`
- 각 `add_count_yearX`는 정수 `>= 0`

## 출력 형식

프로그램은 다음 메시지/형식으로 출력한다.

```text
트리의 깊이를 입력:
초기 상태:
<레벨 1>
<레벨 2>
...

********** Y년 **********
추가하려는 장식의 수:
유지보수 후:
<레벨 1>
<레벨 2>
...
```

각 레벨은 공백 구분으로 한 줄에 출력한다.

## 입출력 예 1

입력:

```text
2
1
1
1
1
1
```

출력(요약):

```text
트리의 깊이를 입력:
초기 상태:
4
9 5

********** 1년 **********
...
유지보수 후:
4
0 5

********** 2년 **********
...
유지보수 후:
5
1 0

... (3~5년차 동일 규칙 반복)
```

## 구현 주의

- 문제에서 제공된 초기 코드의 출력 메시지(`cout`) 순서는 채점 기준에 포함될 수 있다.
- 안내 문자열 형식과 출력 순서를 문제 요구와 동일하게 맞춰야 한다.
