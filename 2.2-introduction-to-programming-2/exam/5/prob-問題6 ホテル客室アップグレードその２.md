# 문제 6: 호텔 객실 업그레이드 2 (구제 기능)

문제 5의 호텔 객실 배정 시뮬레이션에 아래 2가지 규칙을 추가한다.

1. 신규 고객이 통상 배정으로 수용 불가일 때만 "예약 등급 확보(구제)" 시도
2. 해당 밤에 구제가 1회라도 성공하면 그 밤은 업그레이드(Step3) 수행 금지

아래는 문제 5 대비 추가/변경 사항만 설명한다.

## 용어

- 방 이동: 이미 숙박 중인 고객을 다른 방으로 재배정(퇴실 아님)
- 업그레이드 상태 고객: 현재 배정 등급 > 예약 등급
- 예약 등급 확보(구제): 기존 고객 이동으로 신규 고객의 예약 등급 방 1칸을 확보하는 처리

예약 등급이 `B`인 신규 고객은 구제 대상이 아니다.

## 변경된 처리

### A) Step2 신규 고객 수용 판정 확장

통상 배정으로 실패한 신규 고객에 대해:
- 예약 등급이 `P` 또는 `E`이면 구제를 시도
- 구제 성공 시 통상 배정(E->P->B 탐색)을 처음부터 재시도
- 재시도 성공 시 수용
- 재시도 실패 시 미수용

해당 밤에 구제가 한 번이라도 성공하면 `rescueHappened = true`.

### B) Step3 업그레이드 변경

- `rescueHappened = false`: 문제 5와 동일하게 업그레이드 수행
- `rescueHappened = true`: 업그레이드 수행하지 않음

## 구제 상세 규칙

### (1) 예약 P 신규 고객 구제 (P 1칸 확보)

목표: P(201~203)에서 1실 비우기

1. B 공실이 하나도 없으면 실패
2. 후보: 현재 P에 있고 아래 조건 모두 만족
   - 업그레이드 상태
   - 예약등급 B
   - 신규 고객보다 "강한" 고객(status 더 작은 고객)은 제외
3. 후보 없으면 실패
4. 후보가 여러 명이면 "밀어내기 우선순위" 최고 1명 선택
5. 선택 고객을 B 공실 중 가장 큰 방번호로 이동(105 방향 우선)
6. 이동 성공 시 구제 성공

### (2) 예약 E 신규 고객 구제 (E 1칸 확보)

목표: E(301,302)에서 1실 비우기

1. 후보: 현재 E에 있고 아래 조건 모두 만족
   - 업그레이드 상태
   - 예약등급 E가 아님
   - 신규 고객보다 강한 고객(status 더 작은 고객) 제외
2. 후보 없으면 실패
3. 후보가 여러 명이면 밀어내기 우선순위 순으로 시도
4. 후보가 P 숙박 불가면 다음 후보
5. P 공실이 있으면 후보를 P 공실 중 가장 큰 번호로 이동 -> 성공
6. P 만실이면, (1)의 P 구제를 1회 시도
   - 실패: 다음 후보
   - 성공: 확보된 P 공실(가장 큰 번호)에 후보 이동 -> 성공
7. 모든 후보 실패 시 구제 실패

## 밀어내기 우선순위

후보 비교 키(앞 항목 우선):
1. 예약등급이 낮을수록 우선(B -> P -> E)
2. 남은 숙박일(당일 포함)이 적을수록 우선
3. 등록연도가 최근일수록 우선
4. 회원 ID가 작을수록 우선
5. 전체 입력순이 늦을수록 우선

## 구현 대상 함수

초기 코드에서 다음 함수 구현:
- `RollbackMoves`
- `MakeVacancyE`
- `MakeVacancyP`

## 입력

문제 5와 동일.

## 출력

문제 5와 동일.

## 샘플

샘플이 매우 길어 원문 파일의 입력/출력을 참고한다.
