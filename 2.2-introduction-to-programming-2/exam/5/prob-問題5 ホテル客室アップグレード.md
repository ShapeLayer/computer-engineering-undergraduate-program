# 문제 5: 호텔 객실 업그레이드

호텔 객실 배정 시뮬레이션을 수행한다.

객실 구성:
- E(Executive): 301, 302 (2실)
- P(Premium): 201, 202, 203 (3실)
- B(Basic): 101, 102, 103, 104, 105 (5실)

총 10실이다.

각 고객 정보:
- 회원 ID: 9자리 숫자 문자열
  - 앞 4자리: 등록연도
  - 5번째 자리: 회원 status
- 예약 등급: B / P / E
- 숙박일수 `nights` (>=1)

Day `d`에 체크인하면 숙박 밤은 `d`부터 `d+nights-1`까지.

## 밤 t 처리 순서 (t = 1..D)

`D`는 전체 고객의 마지막 숙박 밤 최대값.

### Step1. 연박 고객 유지

전날 배정된 고객 중 밤 `t`에도 숙박하는 고객은 같은 방 유지.

### Step2. 신규 고객 배정 (Day=t 고객)

신규 고객 처리 순서:
1. status 오름차순(0 -> 4)
2. 같은 status에서는 입력 순

각 고객마다:
- 숙박 가능 등급 집합을 회원 규칙으로 판정
- 가능한 등급 중 `E -> P -> B` 순으로 공실 탐색
- 공실 있으면 해당 등급의 가장 작은 방 번호 배정
- 없으면 미배정

### Step3. 업그레이드

신규 배정 후 다음을 반복:

1) E에 공실이 있는 동안
- E 숙박 가능이고 현재 E가 아닌 고객(B/P 숙박) 중
- 업그레이드 우선순위 최고인 1명을 E로 이동

2) P에 공실이 있는 동안
- P 숙박 가능이고 현재 B인 고객 중
- 업그레이드 우선순위 최고인 1명을 P로 이동

## 업그레이드 우선순위

아래 키를 순서대로 비교:
1. status가 작을수록 우선
2. 남은 숙박일(당일 포함)이 적을수록 우선
3. 등록연도가 오래될수록 우선
4. 회원 ID가 클수록 우선
5. 전체 입력순이 빠를수록 우선

## 회원 규칙 (숙박 가능 등급)

공통: 예약 등급보다 낮은 등급에는 숙박 불가.

- status 0,1:
  - 예약 이상 등급 모두 가능
- status 2:
  - 예약 B: B,P 가능 / E 불가
  - 예약 P: P만
  - 예약 E: E만
- status 3:
  - 예약 B:
    - 등록연도 <= 2017: B,P 가능
    - 등록연도 >= 2018: B만
  - 예약 P: P만
  - 예약 E: E만
- status 4:
  - 업그레이드 불가, 예약 등급만 가능

## 구현 대상 함수

초기 코드에서 다음 함수를 완성:
- `CanUseGrade`
- `BuildPrefGradesDesc`

## 입력

Day1~Day5에 대해 순서대로:
- 그날 체크인 신규 수 `nd`
- 이어서 `nd`줄: `id reserveGrade nights`

## 출력

밤 `t=1..D`마다 아래 5줄 출력:
1. `night:t`
2. E행 (301,302 / 나머지 3칸은 `-`)
3. P행 (201,202,203 / 나머지 2칸은 `-`)
4. B행 (101~105)
5. 빈 줄

표기:
- 값은 쉼표 구분
- 공실은 `0`

## 샘플

샘플이 매우 길어 원문 파일의 입력/출력을 참고한다.
