# A + B(rainfxxk)

주어지는 난해한 프로그래밍 언어 「Brainfxxk」의 소스코드의 실행 결과를 작성하여라. 소스코드의 길이 $L$은 $10,000$ 를 넘지 않는다.  

<br />

Brainfxxk는 아래의 규칙에 따라 실행된다:

1. Brainfxxk 인터프리터는 32768개 크기의 `char` 배열과 이 배열의 인덱스를 가리키는 포인터를 메모리 공간으로 가진다. 배열의 모든 요소와 포인터는 `0`으로 초기화된다.
    ```cpp
    // define
    char mem[32768];
    int ptr;

    // init
    memset(mem, 0, sizeof(mem));
    ptr = 0;
    ```
2. 인터프리터는 소스코드의 각 문자를 왼쪽 끝에서부터 오른쪽 방향으로 읽는다. 오른쪽 끝 문자까지 읽었다면, 다음 줄의 왼쪽 끝에서부터 같은 행동을 반복한다.
3. 2 과정에서 읽어들인 개별 문자는 아래와 같이 처리한다.
    - `>` : 포인터 1 증가  
        32768 이상의 메모리 주소는 존재하지 않는다. 메모리 주소 32768은 오버플로우되어 메모리 주소 0이다.
    - `<` : 포인터 1 감소  
        음의 메모리 주소는 존재하지 않는다. 메모리 주소 -1은 언더플로우되어 메모리 주소 32767이다.
    - `+` : 포인터가 가리키는 요소의 값을 1 증가
    - `-` : 포인터가 가리키는 요소의 값을 1 감소
    - `.` : 포인터가 가리키는 요소의 값을 아스키 코드 문자로 출력
    - `,` : 표준입력으로부터 값을 입력받아, 그 값의 아스키 코드를 포인터가 가리키는 요소에 덮어씀
    - `[` - 포인터가 가리키는 바이트의 값이 0이라면 짝이 되는 `]`로 이동
    - `]` - 포인터가 가리키는 바이트의 값이 0이 아니면 짝이 되는 `[`로 이동
    - `\0` : EOF(End of File); 프로그램을 종료한다.
    - 실제 Brainfxxk과 달리, 이 문제에서는 `,` 문자는 무시한다.
    - 위 정의를 제외한 모든 문자는 무시하고 계속 진행한다.

인터프리터는 EOF(`\0`; 아스키코드 0)를 입력받기 전까지는 계속해서 입력을 받아와야 한다. 만약 로컬에서 답안을 테스트 실행하는 도중, 파이프라인¹을 사용하지 않고 직접 표준입력에 값을 입력하고자 한다면, EOF 문자를 직접 입력하여야 한다.

- 윈도우: `Ctrl` + `Z`를 입력하고 `Enter` 한다.
- 리눅스/mac: `Ctrl` + `D`를 입력하고 `Enter` 한다.

이 과정에서 표준 입력에 입력으로 나타나는 `^Z`와 `^D`는 실제 `^Z`, `^D`라는 문자열이 아닌, `Ctrl`과 `Z` 혹은 `D`를 조합해 입력한 문자라는 의미이다. 

¹ `cat <input> | <compiled-program>`

<br />

아래는 몇 가지 Brainfxxk 예제이다.

<br />

예제 입력 1

```brainfuck
+++++++++++++++++++++++++++++++++.
```

예제 출력 1
```
!
```

예제 1에서 `+`가 33개 등장하여 0번 메모리 공간의 값이 33이 되었다. `.`에 의해 아스키코드 33이 나타내는 문자, `!`가 출력되었다.

<br />

예제 입력 2

```brainfuck
+++++[>++++++++++<-]>.
```

예제 출력 2

```
2
```

1. 앞의 `+++++`로 0번 메모리 공간의 값이 5가 되었다. `[`를 처리할 때, 포인터가 가리키는 0번 메모리 공간의 값이 0이 아니므로 `]`로 건너뛰지 않고 계속 진행한다.
2. `[` 다음의 `>`에 의해 포인터가 1 증가하여, 이후부터는 1번 메모리 공간을 사용한다.
3. `>` 다음에 `+`가 10개 등장하여, 1번 메모리의 값은 10이 되었다.
4. `+` 다음의 `<`에 의해 포인터가 1 감소하여, 이후부터는 0번 메모리 공간을 사용한다.
5. `<` 다음의 `-`에 의해 0번 메모리 공간의 값이 1 감소한 4가 되었다.
6. `]`를 처리할 때, 포인터가 가리키는 0번 메모리 공간의 값이 0이 아니므로, 계속 진행하지 않고 `[`로 되돌아간다. (`mem = { 4, 10, ... };`)
7. 2 ~ 6 과정을 4회 더 반복하여 `mem = { 0, 50, ... };`이 된다.
8. `]`를 처리할 때, 포인터가 가리키는 0번 메모리 공간의 값이 0이므로, 더 이상 되돌아가지 않고 다음 문자를 처리한다.
9. `>`에 의해 포인터가 1 증가하여, 1번 메모리 공간을 사용한다.
10. `.`에 의해 아스키코드 50이 나타내는 문자, `2`가 출력된다.

## 입력

첫째 줄에 brainfxxk의 소스코드가 주어진다.

## 출력

brainfxxk 코드를 실행하였을 때 표준 출력 되는 값을 출력하여라.

# 日本語

「Brainfxxk」という難解なプログラミング言語のソースコードの実行結果を記述してください。ソースコードの長さ$L$は$10,000$を超えません。

<br />

Brainfxxkは以下の規則に従って実行されます。

1.  Brainfxxkインタプリタは、32768個の`char`配列と、この配列のインデックスを指すポインタをメモリ空間として持ちます。配列のすべての要素とポインタは`0`で初期化されます。
    ```cpp
    // define
    char mem[32768];
    int ptr;

    // init
    memset(mem, 0, sizeof(mem));
    ptr = 0;
    ```
2.  インタプリタは、ソースコードの各文字を左端から右方向へ読み込みます。右端の文字まで読み込んだら、次の行の左端から同じ動作を繰り返します。
3.  2の過程で読み込まれた個々の文字は、以下のように処理されます。
    -   `>` : ポインタを1増加させます。
        32768以上のメモリアドレスは存在しません。メモリアドレス32768はオーバーフローしてメモリアドレス0になります。
    -   `<` : ポインタを1減少させます。
        負のメモリアドレスは存在しません。メモリアドレス-1はアンダーフローしてメモリアドレス32767になります。
    -   `+` : ポインタが指す要素の値を1増加させます。
    -   `-` : ポインタが指す要素の値を1減少させます。
    -   `.` : ポインタが指す要素の値をASCIIコード文字として出力します。
    -   `,` : 標準入力から値を読み込み、その値のASCIIコードをポインタが指す要素に上書きします。
    -   `[` - ポインタが指すバイトの値が0の場合、対応する`]`に移動します。
    -   `]` - ポインタが指すバイトの値が0でない場合、対応する`[`に戻ります。
    -   `\0` : EOF (End of File); プログラムを終了します。
    -   実際のBrainfxxkとは異なり、この問題では`,`文字は無視されます。
    -   上記の定義を除くすべての文字は無視され、処理は続行されます。

インタプリタはEOF (`\0`; ASCIIコード0) を受け取るまで、入力を受け取り続ける必要があります。もしローカルで解答を実行する際に、パイプライン¹を使用せずに直接標準入力に値を入力したい場合は、EOF文字を直接入力する必要があります。

-   Windows: `Ctrl` + `Z`を入力し、`Enter`キーを押します。
-   Linux/macOS: `Ctrl` + `D`を入力し、`Enter`キーを押します。

この過程で標準入力に表示される`^Z`や`^D`は、実際の`^Z`、`^D`という文字列ではなく、`Ctrl`と`Z`または`D`を組み合わせて入力した文字を意味します。

¹ `cat <input> | <compiled-program>`　(ex. `cat input.txt | ./a.out`)

<br />

以下にいくつかのBrainfxxkの例を示します。

<br />

例1

```brainfuck
+++++++++++++++++++++++++++++++++.
```

例1の出力
```
!
```

例1では、`+`が33個登場し、0番メモリ空間の値が33になりました。`.`によってASCIIコード33が表す文字`!`が出力されました。

<br />

例2

```brainfuck
+++++[>++++++++++<-]>.
```

例2の出力

```
2
```

1.  最初の`+++++`により、0番メモリ空間の値が5になりました。`[`を処理する際、ポインタが指す0番メモリ空間の値が0ではないため、`]`にスキップせずに処理を続行します。
2.  `[`の次の`>`によりポインタが1増加し、以降は1番メモリ空間を使用します。
3.  `>`の次に`+`が10個登場し、1番メモリの値は10になりました。
4.  `+`の次の`<`によりポインタが1減少し、以降は0番メモリ空間を使用します。
5.  `<`の次の`-`により、0番メモリ空間の値が1減少し4になりました。
6.  `]`を処理する際、ポインタが指す0番メモリ空間の値が0ではないため、これ以上進まずに`[`に戻ります。(mem = { 4, 10, ... };)
7.  2〜6の過程をさらに4回繰り返し、`mem = { 0, 50, ... };`となります。
8.  `]`を処理する際、ポインタが指す0番メモリ空間の値が0であるため、これ以上戻らずに次の文字を処理します。
9.  `>`によりポインタが1増加し、1番メモリ空間を使用します。
10. `.`によりASCIIコード50が表す文字`2`が出力されます。

## 入力
最初の行にBrainfuckのソースコードが与えられます。

## 出力
Brainfuckコードを実行した際に標準出力される値を出力してください。
