「Brainfxxk」という難解なプログラミング言語のソースコードの実行結果を記述してください。ソースコードの長さ$$$L$$$は$$$10,000$$$を超えません。

<br />

Brainfxxkは以下の規則に従って実行されます。

  1. Brainfxxkインタプリタは、32768個のchar配列と、この配列のインデックスを指すポインタをメモリ空間として持ちます。配列のすべての要素とポインタは0で初期化されます。

    // define
    char mem[32768];
    int ptr;

    // init
    memset(mem, 0, sizeof(mem));
    ptr = 0;

  2. インタプリタは、ソースコードの各文字を左端から右方向へ読み込みます。右端の文字まで読み込んだら、次の行の左端から同じ動作を繰り返します。

  3. 2の過程で読み込まれた個々の文字は、以下のように処理されます。

    • > : ポインタを1増加させます。 32768以上のメモリアドレスは存在しません。メモリアドレス32768はオーバーフローしてメモリアドレス0になります。

    • < : ポインタを1減少させます。 負のメモリアドレスは存在しません。メモリアドレス-1はアンダーフローしてメモリアドレス32767になります。

    • + : ポインタが指す要素の値を1増加させます。

    • - : ポインタが指す要素の値を1減少させます。

    • . : ポインタが指す要素の値をASCIIコード文字として出力します。

    • , : 標準入力から値を読み込み、その値のASCIIコードをポインタが指す要素に上書きします。

    • [ - ポインタが指すバイトの値が0の場合、対応する]に移動します。

    • ] - ポインタが指すバイトの値が0でない場合、対応する[に戻ります。

    • 0 : EOF (End of File); プログラムを終了します。

    • 実際のBrainfxxkとは異なり、この問題では,文字は無視されます。

    • 上記の定義を除くすべての文字は無視され、処理は続行されます。

インタプリタはEOF (0; ASCIIコード0) を受け取るまで、入力を受け取り続ける必要があります。もしローカルで解答を実行する際に、パイプライン¹を使用せずに直接標準入力に値を入力したい場合は、EOF文字を直接入力する必要があります。

  • Windows: Ctrl + Zを入力し、Enterキーを押します。

  • Linux/macOS: Ctrl + Dを入力し、Enterキーを押します。

この過程で標準入力に表示される^Zや^Dは、実際の^Z、^Dという文字列ではなく、CtrlとZまたはDを組み合わせて入力した文字を意味します。

¹ cat <input> | <compiled-program> (ex. cat input.txt | ./a.out)

<br />

例2

+++++[>++++++++++<-]>.

例2の出力

2
  1. 最初の+++++により、0番メモリ空間の値が5になりました。[を処理する際、ポインタが指す0番メモリ空間の値が0ではないため、]にスキップせずに処理を続行します。

  2. [の次の>によりポインタが1増加し、以降は1番メモリ空間を使用します。

  3. >の次に+が10個登場し、1番メモリの値は10になりました。

  4. +の次の<によりポインタが1減少し、以降は0番メモリ空間を使用します。

  5. <の次の-により、0番メモリ空間の値が1減少し4になりました。

  6. ]を処理する際、ポインタが指す0番メモリ空間の値が0ではないため、これ以上進まずに[に戻ります。(mem = { 4, 10, ... };)

  7. 2〜6の過程をさらに4回繰り返し、mem = { 0, 50, ... };となります。

  8. ]を処理する際、ポインタが指す0番メモリ空間の値が0であるため、これ以上戻らずに次の文字を処理します。

  9. >によりポインタが1増加し、1番メモリ空間を使用します。

  10. .によりASCIIコード50が表す文字2が出力されます。
