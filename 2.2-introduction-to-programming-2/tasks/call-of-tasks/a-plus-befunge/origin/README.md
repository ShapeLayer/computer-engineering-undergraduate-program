# A + B(efunge)

난해한 프로그래밍 언어 「Befunge」는 아래와 같은 규칙을 가진다.

 - `0` ~ `9` : 해당 수를 넣는다.
 - `+` : a, b를 꺼내 a + b의 값을 넣는다.
 - `-` : a, b를 꺼내 b - a의 값을 넣는다.
 - `*` : a, b를 꺼내 a * b의 값을 넣는다.
 - `/` : a, b를 꺼내 b ÷ a의 몫을 넣는다. 만약 먼저 나온 수가 0이면 연산 결과를 사용자가 직접 입력하여 넣는다.
 - `%` : a, b를 꺼내 뒤에 b ÷ a 의 나머지를 넣는다. 만약 먼저 나온 수가 0이면 연산 결과를 사용자가 직접 입력하여 넣는다.
 - `!` : 꺼낸 수가 0이면 1, 아니면 0을 넣는다.
 - `` ` `` : a, b를 꺼내 b > a면 1, 아니면 0을 넣는다.
 - `>` : 명령 포인터를 오른쪽으로 이동하게 한다.
 - `<` : 명령 포인터를 왼쪽으로 이동하게 한다.
 - `^` : 명령 포인터를 위로 이동하게 한다.
 - `v` : 명령 포인터를 아래로 이동하게 한다.
 - `?` : 명령 포인터를 랜덤한 방향으로 이동하게 한다.
 - `_` : 꺼낸 숫자가 0이면 명령 포인터를 오른쪽, 아니면 왼쪽으로 이동하게 한다.
 - `|` : 꺼낸 숫자가 0이면 명령 포인터를 아래, 아니면 위로 이동하게 한다.
 - `"` : 다음 `"`가 등장할 때까지 문자를 넣는다.
 - `:` : 스택의 가장 위의 값을 복사해 넣는다.
 - `\` : 스택의 가장 위의 두 값의 위치를 서로 바꾼다.
 - `$` : 스택에서 숫자를 꺼낸다.
 - `.` : 숫자를 꺼내 출력한다.
 - `,` : 숫자를 꺼내 ASCII 문자를 출력한다.
 - `#` : 다음 명령을 건너 뛴다.
 - `g` : y, x를 꺼내 명령의 (x, y)위치에 있는 문자의 ASCII값을 넣는다.
 - `p` : y, x, v를 꺼내 명령의 (x, y)위치에 있는 문자를 v에 해당하는 ASCII 문자로 바꾼다.
 - `&` : 숫자를 입력받아 넣는다.
 - `~` : 문자를 입력받아 넣는다.
 - `@` : 프로그램을 종료한다.
 - `&`와 `~`는 본 문제에서는 무시한다.

두 수 $A$와 $B$, 그리고 Befunge 소스코드의 가로 길이 $w$와 세로 길이 $h$가 주어질 때, $A$와 $B$, 그리고 $A+B$를 공백으로 구분하여 출력하는 $w \times h$ 크기의 Befunge 소스코드를 생성하는 프로그램을 작성하여라. $(0 \le A, B \le 50, 1 \le w, h \le 20)$  

프로그램이 생성한 Befunge 코드를 실행하면 전체 칸의 절반 이상은 최소 한 번 실행되어야 한다. (전체 칸의 절반 이상은 커서가 거쳐가야한다.)  

만약 그 어떤 경우에도 Befunge 코드를 생성할 수 없다면, `IMPOSSIBLE`을 출력하여라.

스페셜 저지: 이 문제는 채점 서버 측의 채점 프로그램이 직접 소스코드를 컴파일하여 제출 답안을 평가하므로, 채점 서버의 정해 프로그램의 출력이 같을 필요는 없다.

아래는 $A=27$, $B=3$, $w=12$, $h=10$ 일 때, $A$, $B$, $A+B$를 공백으로 구분한 `27 3 30`를 출력하는 $12 \times 9 = 108$ 칸 크기의 Befunge 프로그램 코드이다. 하지만 전체 $108$개 칸 중 $42$개 칸만 커서가 거쳐가므로 채점 프로그램은 오답으로 처리할 것이다.  

```befunge
v           
>8439*.*,  v
v    ,*.384<
3           
7           
3           
+           
*           
>        . @
```

아래는 커서가 $75$개 칸을 거쳐가므로 채점 프로그램은 정답으로 처리할 것이다.

```befunge
v           
>8439*.*,  v
v    ,*.384<
>37        v
            
3          <
+           
*           
>        . @
```

# 日本語

# A + B(Befunge)

難解なプログラミング言語「Befunge」は以下の規則を持ちます。

- `0` ~ `9` : その数字をスタックにプッシュします。
- `+` : a, bをポップして a + b の値をプッシュします。
- `-` : a, bをポップして b - a の値をプッシュします。
- `*` : a, bをポップして a * b の値をプッシュします。
- `/` : a, bをポップして b ÷ a の商をプッシュします。もし先にポップされた数(a)が0の場合、演算結果をユーザーが直接入力してプッシュします。
- `%` : a, bをポップして b ÷ a の剰余をプッシュします。もし先にポップされた数(a)が0の場合、演算結果をユーザーが直接入力してプッシュします。
- `!` : ポップした数が0であれば1、そうでなければ0をプッシュします。
- `` ` `` : a, bをポップして b > a であれば1、そうでなければ0をプッシュします。
- `>` : 命令ポインタを右に移動させます。
- `<` : 命令ポインタを左に移動させます。
- `^` : 命令ポインタを上に移動させます。
- `v` : 命令ポインタを下に移動させます。
- `?` : 命令ポインタをランダムな方向に移動させます。
- `_` : ポップした数字が0であれば命令ポインタを右に、そうでなければ左に移動させます。
- `|` : ポップした数字が0であれば命令ポインタを下に、そうでなければ上に移動させます。
- `"` : 次の「"」が出現するまで、文字のASCII値をプッシュします。
- `:` : スタックの最上位の値をコピーしてプッシュします。
- `\` : スタックの最上位の2つの値の位置を入れ替えます。
- `$` : スタックから数字をポップします。(破棄します)
- `.` : 数字をポップして出力します。
- `,` : 数字をポップしてASCII文字として出力します。
- `#` : 次の命令をスキップします。
- `g` : y, xをポップして、命令の (x, y)位置にある文字のASCII値をプッシュします。
- `p` : y, x, vをポップして、命令の (x, y)位置にある文字を、vに該当するASCII文字に書き換えます。
- `&` : 数字を入力として受け取り、プッシュします。
- `~` : 文字を入力として受け取り、プッシュします。
- `@` : プログラムを終了します。
- `&`と`~`は本問題では無視されます。

2つの数 $A$と$B$、そしてBefungeソースコードの横幅 $w$と縦幅 $h$が与えられたとき、$A$と$B$、そして$A+B$をスペースで区切って出力する $w \times h$ サイズのBefungeソースコードを生成するプログラムを作成してください。$(0 \le A, B \le 50, 1 \le w, h \le 20)$

プログラムが生成したBefungeコードを実行すると、全体セルの半分以上は少なくとも一度実行される必要があります。(全体セルの半分以上をカーソルが通過する必要があります。)

もし、いかなる場合においてもBefungeコードを生成できない場合は、「IMPOSSIBLE」と出力してください。

スペシャルジャッジ: この問題は採点サーバー側の採点プログラムが直接ソースコードをコンパイルして提出された解答を評価するため、採点サーバーの正解プログラムの出力と同じである必要はありません。

以下は、$A=27$、$B=3$、$w=12$、$h=10$ の場合において、$A$、$B$、そして$A+B$をスペースで区切った「27 3 30」を出力する $12 \times 9 = 108$ セルサイズのBefungeプログラムコードです。しかし、全体の108セル中42セルしかカーソルが通過しないため、採点プログラムは不正解と判断します。

```befunge
v           
>8439*.*,  v
v    ,*.384<
3           
7           
3           
+           
*           
>        . @
```

以下はカーソルが75セルを通過するため、採点プログラムは正解と判断します。

```befunge
v           
>8439*.*,  v
v    ,*.384<
>37        v
            
3          <
+           
*           
>        . @
```
